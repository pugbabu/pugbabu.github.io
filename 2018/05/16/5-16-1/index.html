用Vue开发电商页面时，电梯容器功能中遇到的一些问题

2018-05-16
前端
Vue

大半个月没有更新博客，上周从泰一指尚离职，换了家做电商平台的新公司，接手了离职小伙伴的活，继续开发页面各组件。

项目梳理

整个项目用vue-cli搭建完成，查看package.json中的项目依赖，主要模块有：

  • axios 用于请求后台接口
  • muse-ui 一个基于 Vue 2.0 和 Material Design 的 UI 组件库https://github.com/museui/muse-ui项目中用的不是特别多
  • vue 项目主要框架
  • vuex 项目中有引入，但是之前开发人员没有使用，接手后将项目中多个层级通信改用vuex
  • vue-awesome-swiper 用于图片轮播组件
  • vue-router 路由
  • vue-lazyload 图片懒加载，暂未看到项目中使用，着手优化商品列表时给图片加上v-lazy

任务梳理

完成2个页面：首页、活动页，每个页面渲染的组件都是在后台配置的，后台返回需要渲染组件的id和type，如

拿到type值，渲染对应的组件

实现思路

因为页面中的组件是动态的，组件内容和位置都不固定，所以新建一个vue文件，里面放入所有可能会用到的组件

然后在首页或者活动页中循环渲染组件

考虑到页面性能，初始时渲染9个组件，滑到底部时再加载3个组件，该功能利用muse-ui的加载组件完成

遇到的问题

1）电梯容器，点击tab，页面定位到相应组件位置，由于电梯容器组件和tab对应页面组件同属ModulesBox的子组件，并且是v-for循环生成的，点击tab后，滚动功能报错，原因在于：1、电梯容器节点未找到，获取电梯容器位置时出错；2、tab点击对应的组件未找到
解决方法：
不得不说，vue的watch钩子真的很好用，尤其是解决dom节点找不到这种恶心人不偿命的杂症。首先，监听渲染组件数组loadList，如果数组中已经有了电梯容器组件时，获得电梯容器节点，并保存电梯容器的offsetTop值

点击电梯容器的tab时，更新store的floorId值，然后在首页或者活动页中监听floorId值,存在该值时执行滚动动作

2）点击电梯容器tab，滚动到对应位置的问题解决了，接下来再开发滑动页面时，当滑到电梯容器某tab对应的组件时，该tab需变更为激活状态，这过程中又出现了节点找不到的问题
解决方法：
在开发电梯容器功能时，监听loadList，判断电器容器存在时，执行了一个计算高度的方法this._calculateHeight()，通过该方法，获得每个组件的offsetTop值，通过该值，和页面滑动值做比较

然后监听floorTop值（电梯容器在页面中的位置），如果存在时，window.addEventListener(‘scroll’, this.handleScroll)

在handleScroll方法中，处理scroll值和组件容器位置值的对比

目前功能基本满足需求，但还是有些小问题，需要继续完善。
